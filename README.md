# FraudGuard — Credit Card Anomaly Detection API

A production-ready REST API for detecting fraudulent credit card transactions using an **Isolation Forest** model trained on the [Kaggle creditcard dataset](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud).

## Features

- **Unsupervised anomaly detection** — no labelled fraud needed at inference time
- **Score + label + explanation** — each prediction includes an anomaly score, FRAUD/NORMAL label, confidence, and the top 5 contributing features
- **Batch prediction** — up to 500 transactions per request
- **Railway-ready** — `railway.toml` and `Dockerfile` included

## Quickstart

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Train the model

```bash
python -m app.train
# Optional flags:
# --data path/to/creditcard.csv
# --contamination 0.002   (expected fraud rate, default 0.2%)
```

This saves three files to `models/`:
- `isolation_forest.pkl` — trained model
- `scaler.pkl` — StandardScaler fitted on training data
- `model_meta.json` — version, threshold, feature info

### 3. Run the API

```bash
uvicorn app.main:app --reload
```

Docs at: http://localhost:8000/docs

### 4. Run tests

```bash
pytest tests/ -v
```

## API Reference

### `POST /predict`

Analyse a single transaction.

```json
// Request
{
  "Time": 406.0,
  "V1": -2.312, "V2": 1.952, "...",
  "Amount": 149.62
}

// Response
{
  "is_fraud": true,
  "label": "FRAUD",
  "anomaly_score": -0.082341,
  "confidence": 0.582,
  "explanation": {
    "top_features": ["V14", "V4", "V12", "V10", "V17"],
    "feature_contributions": {
      "V14": -4.289,
      "V4": 3.997,
      "V12": -2.9,
      "V10": -2.772,
      "V17": -2.83
    }
  },
  "model_version": "v1713456789"
}
```

### `POST /predict/batch`

Analyse up to 500 transactions.

```json
// Request
{ "transactions": [ {...}, {...} ] }

// Response
{
  "results": [...],
  "total": 2,
  "fraud_count": 1,
  "normal_count": 1
}
```

### `GET /health`

```json
{ "status": "ok", "model_loaded": true, "model_version": "v1713456789", "training_samples": 284315 }
```

### `GET /model/info`

Returns model hyperparameters and training metadata.

## Deployment on Railway

1. Train the model locally first: `python -m app.train`
2. Commit the `models/` directory (the three generated files)
3. Push to your Railway-linked GitHub repo

Railway will build the Dockerfile and use `railway.toml` for the start command and health check.

> **Note:** `models/` is intentionally committed so Railway doesn't need to retrain on every deploy. For a production setup, consider storing model artifacts in object storage (e.g. S3 or Supabase Storage) and loading them at startup.

## Model Details

| Setting | Value |
|---|---|
| Algorithm | Isolation Forest |
| n_estimators | 200 |
| contamination | 0.002 (0.2%) |
| Features | 30 (Time, V1–V28, Amount) |
| Training data | Normal transactions only |
| Threshold | 1st percentile of training scores |

The model is trained **only on normal transactions** — the standard unsupervised approach. It learns the distribution of legitimate behaviour, then flags transactions that don't fit.
